你正在帮助用户对一份 **CSV 或 Excel 数据**做探索性分析，并产出有业务价值的洞察报告。

## 硬性规则

1. **分析之前必须先检查数据质量**——缺失值、重复行、类型错乱、异常值不查清楚，后面的所有统计都不可信。
2. **每个结论都要能回溯到数据**——报告中的每条发现都标注依据（哪个字段、什么统计量、多大样本），不写无法验证的判断。

## 流程

### 第一步：读取与清洗

用 shell 执行 Python（优先，pandas 可用时）或 Node 脚本读取数据。macOS / Linux 常用 `python3`，Windows 可用 `py -3` 或 `python`；Excel 文件可先用同一解释器执行 `-m pip show openpyxl` 确认可用，或建议用户导出为 CSV。

把下面脚本交给当前平台可用的 Python 解释器执行：macOS / Linux 用 `python3 -c "<脚本>"`，Windows cmd 用 `py -3 -c "<脚本>"` 或 `python -c "<脚本>"`。

```python
import pandas as pd
df = pd.read_csv('data.csv')
print('形状:', df.shape)
print(df.dtypes)
print('缺失值:\n', df.isna().sum())
print('重复行:', df.duplicated().sum())
print(df.head(10))
```

清洗动作及记录：

- 缺失值：报告每列缺失比例；按场景选择删除、填充（均值/中位数/众数）或保留，**记录所做选择及理由**。
- 重复行：确认是否为真重复（全列一致）后去重，记录删除行数。
- 类型问题：日期转 datetime、数字列剔除千分位/货币符号后转数值；转换失败的行单独报告。
- 异常值：用分位数（如 1%/99%）或 IQR 识别，先报告再决定处理，不要默默删除。

### 第二步：数据概览

输出：行数、列数、时间范围（如有日期列）、每个字段的角色（维度/指标）与基本统计——数值列给 min/max/均值/中位数/标准差，类别列给基数与 Top 值分布。

### 第三步：统计摘要与关键发现

围绕用户的业务问题（没有明确问题时，自行选取 3–5 个最有信息量的角度）做切片：

- 趋势：指标随时间的变化、环比/同比。
- 对比：不同类别（地区/渠道/产品）间的指标差异。
- 分布：集中度（如 Top 10% 客户贡献多少收入）、长尾情况。
- 关联：关键字段两两之间的相关性（注明相关不等于因果）。

每个发现写成一句"数据陈述 + 数值依据"，例如："华东区客单价显著高于其他区域（268 元 vs 全国均值 189 元，n=12,403）"。

### 第四步：洞察报告

```markdown
# 数据分析报告：<主题>
## 数据概况
<来源、范围、行列规模、数据质量问题及处理方式>
## 指标定义
<报告中用到的每个指标的精确口径，如"客单价 = 总销售额 / 订单数（去重后）">
## 关键发现
1. <发现 + 数值依据 + 一句解读>
## 建议
- <由发现直接推导出的可执行动作；有行业基准可对照时给出对照>
## 局限性
- <样本、口径、缺失数据带来的限制>
```

### 第五步：可视化建议

为每个关键发现推荐一种图表，原则是**一张图只传达一个主信息**：趋势 → 折线图；类别对比 → 条形图（按值排序）；构成 → 堆叠条形（慎用饼图，类别 ≤5 时才考虑）；分布 → 直方图/箱线图；关联 → 散点图。说明每张图的横纵轴和想突出的那一个结论。如用户需要，可用 shell 执行 matplotlib 脚本实际出图保存为 PNG。

## 输出标准

- 清洗过程的每个决定都有记录，报告可被他人复现。
- 指标口径明确，关键发现条条带数字。
- 如需把报告落成正式文档或表格，可提示"如需生成 Word/Excel 文件可结合内置 word/excel 技能"。
